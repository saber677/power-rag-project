# 订单系统

## 订单状态流转

订单状态包括：待支付、已支付、待发货、已发货、已签收、已完成、已取消、退款中、已退款。

状态流转规则：
- 待支付 → 已支付（用户完成支付）
- 待支付 → 已取消（超时未支付，默认30分钟）
- 已支付 → 待发货（系统自动流转）
- 待发货 → 已发货（仓库发货）
- 已发货 → 已签收（物流签收）
- 已签收 → 已完成（确认收货或7天自动确认）
- 已支付/待发货 → 退款中（用户申请退款）
- 退款中 → 已退款（审核通过）
- 退款中 → 已支付（审核拒绝，恢复原状态）

## 订单取消逻辑

用户可主动取消订单的条件：
- 订单状态为待支付
- 订单状态为已支付且未发货（待发货状态）

系统自动取消：
- 待支付订单超过30分钟未支付，系统自动取消
- 定时任务每分钟扫描一次过期订单

取消后处理：
- 释放库存锁定
- 如已支付，发起退款到原支付渠道
- 发送取消通知（站内信 + 短信）

## 订单拆单规则

当订单中的商品满足以下条件时触发拆单：
1. 商品属于不同仓库
2. 商品属于不同供应商
3. 预售商品与现货商品混合

拆单后每个子订单独立发货、独立跟踪物流。
