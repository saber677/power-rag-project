# 库存系统

## 库存类型

- 可用库存：可正常售卖的数量
- 锁定库存：已下单未支付/未发货的占用数量
- 在途库存：采购已发货但未入库的数量

关系：实际库存 = 可用库存 + 锁定库存

## 库存扣减策略

采用"下单锁定，支付确认"模式：
1. 用户下单 → 锁定库存（可用库存减少，锁定库存增加）
2. 支付成功 → 确认扣减（锁定库存减少）
3. 订单取消 → 释放库存（锁定库存减少，可用库存恢复）

## 超卖防护

- Redis 预扣库存，保证高并发下不超卖
- 数据库乐观锁兜底（version 字段）
- 库存不足时返回"库存不足"提示，不允许下单

## 库存预警

- 当可用库存 ≤ 预警阈值时，自动发送补货通知
- 默认预警阈值：10件
- 通知方式：钉钉群机器人 + 邮件

## 盘点流程

1. 创建盘点单，选择仓库和商品范围
2. 仓库人员实际清点并录入数量
3. 系统对比账面库存与实际库存
4. 差异审批后调整库存
